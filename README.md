# FAPI_KR1 - Контрольная работа №1 по FastAPI

Решение контрольной работы по дисциплине "Технологии разработки серверных приложений" (ИПТИП, 4 семестр 2025/2026).

## Структура проекта

```
FAPI_KR1/
├── app.py              # Основное FastAPI приложение
├── models.py           # Pydantic модели для валидации данных
├── index.html          # HTML страница (Задание 1.2)
├── requirements.txt    # Зависимости Python
├── .gitignore          # Git конфигурация
└── README.md           # Этот файл
```

## Установка и запуск

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск приложения

```bash
uvicorn app:main_app --reload
```

Приложение будет доступно по адресу: `http://localhost:8000`

### 3. Документация API

- Интерактивная документация (Swagger UI): `http://localhost:8000/docs`
- Альтернативная документация (ReDoc): `http://localhost:8000/redoc`

## Реализованные задания

### Задание 1.1 ✅
**Простое JSON приложение**
- Маршрут: `GET /`
- Возвращает: `{"message": "Авторелоад действительно работает"}`
- Включена проверка авторелоада при изменении кода

### Задание 1.2 ✅
**Возврат HTML страницы**
- Маршрут: `GET /html`
- Возвращает: HTML файл `index.html` с текстом "Я ОБОЖАЮ ВСТАВАТЬ К ПЕРВОЙ ПАРЕ :)"

### Задание 1.3 ✅
**Сумма двух чисел**
- Маршрут: `POST /calculate`
- Параметры: `num1` (float), `num2` (float)
- Возвращает: `{"result": сумма}`
- Пример: `/calculate?num1=5&num2=10` → `{"result": 15}`

### Задание 1.4 ✅
**Модель пользователя (User)**
- Маршрут: `GET /users`
- Возвращает: `{"name": "Дарья Невская", "id": 1}`
- Использует Pydantic модель `User` из `models.py`

### Задание 1.5 ✅
**Проверка совершеннолетия пользователя**
- Маршрут: `POST /user`
- Принимает JSON: `{"name": "Имя", "age": возраст}`
- Возвращает: `{"name": "Имя", "age": возраст, "is_adult": true/false}`
- Пример:
  - Входные данные: `{"name": "Артур Дворецкий", "age": 26}`
  - Ответ: `{"name": "Артур Дворецкий", "age": 26, "is_adult": true}`

### Задание 2.1 ✅
**Отправка отзывов (Feedback)**
- Маршрут: `POST /feedback`
- Принимает JSON: `{"name": "Имя", "message": "Текст отзыва"}`
- Сохраняет отзывы в памяти
- Возвращает: `{"message": "Спасибо, Имя! Ваш отзыв сохранён."}`

### Задание 2.2 ✅
**Валидация отзывов с проверкой запретных слов**
- Маршрут: `POST /feedback`
- Валидация поля `name`: 2-50 символов
- Валидация поля `message`: 10-500 символов
- Проверка на запретные слова: "кринж", "рофл", "вайб" (в любых падежах)
- При обнаружении запретного слова возвращает HTTP 422 с ошибкой валидации
- Пример ошибки:
  ```json
  {
    "detail": [
      {
        "type": "value_error",
        "loc": ["body", "message"],
        "msg": "Value error, Использование недопустимых слов",
        "input": "Какой-то кринж...",
        "ctx": {"error": {}}
      }
    ]
  }
  ```

## Дополнительные маршруты

### GET /feedbacks
- Просмотр всех сохранённых отзывов
- Возвращает: список отзывов и их количество

## Примеры тестирования
